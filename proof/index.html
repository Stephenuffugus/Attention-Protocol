How It Works
From user interaction to cryptographic proof in milliseconds.
1

User interacts with any digital content

Reading a policy, watching a video, taking a survey, playing a game — any interaction on any device.

Timing entropy, Fitts' Law, Hick's Law, scroll saccade, micro-pause, touch variance, keystroke dynamics, reading speed, hover dwell, tab visibility, inactivity gaps — all running silently in the background.

The attention payload is cryptographically hashed. The receipt is verifiable, portable, and impossible to forge. Store it now, verify it later.

Works offline-first. Hashes stored in localStorage immediately, synced to Firestore when connected. No data loss, no dependency on network.
